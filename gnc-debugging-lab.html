GNC Debugging Lab

This lab is designed as an interactive engineering exercise. Instead of only reading a solved problem, the student tests an incorrect model, observes what fails, applies the correct fix, and checks the result.

The first version focuses on two core ideas from a simplified 2D spacecraft simulation: rotational-state propagation and spacecraft state-bus consistency.

Try → Observe → Diagnose → Fix → Validate → Learn

What to notice

If angular acceleration is integrated directly into orientation, the model skips angular velocity. That means the rotational state is not physically complete.

Correct rotational chain:

\( \dot{\omega} = \alpha \)
\( \dot{\theta} = \omega \)
Stage 2 — SatelliteBus Checker

Problem

A controller can only use the states that are exposed to it. In a spacecraft simulation, the propagated state is often passed through a structured bus. If the bus is missing a key variable, the controller may fail even if the internal dynamics are correct.

Select the fields included in the spacecraft state bus.

Select bus fields and validate the spacecraft state.

Engineering lesson

For attitude control, orientation alone is not enough. A PD controller also needs angular velocity so the derivative term can damp the response.

Required attitude state:

\( \text{state}_{attitude} = [\theta,\ \omega] \)
